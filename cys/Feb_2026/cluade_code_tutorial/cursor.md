## 快速开始

### 前置要求

- Cursor IDE 已安装
- Anthropic API Key（从咸鱼购买获取）

## Cursor 配置 Claude API

### 步骤 1：获取 Anthropic API Key
复制保存你的 API Key（格式：`sk-ant-xxx...`）

> **提示**：API Key 只显示一次，请妥善保管

### 步骤 2：在 Cursor 中配置

1. 打开 Cursor IDE
2. 按 `Cmd+,`（macOS）或 `Ctrl+,`（Windows/Linux）打开设置
3. 在左侧导航栏选择 **Models**（模型）
4. 找到 **API Keys** 部分
5. 在 **Anthropic API Key** 输入框中粘贴你的 API Key
6. 点击保存

### 步骤 3：选择模型

配置完成后，你可以在以下场景中选择使用 Claude：

| 位置 | 说明 |
|------|------|
| **Chat 面板** | 点击模型选择器，选择 Claude Sonnet / Opus |
| **Inline Edit** | 使用 `Cmd+K` / `Ctrl+K` 时自动使用配置的模型 |
| **Composer** | `Cmd+I` / `Ctrl+I` 打开的大面板 |
| **Agent 模式** | `Cmd+Shift+R` / `Ctrl+Shift+R` 触发自主编辑 |

### 可用的 Claude 模型

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| **Claude Haiku 4** | 快速响应，成本低 | 简单代码补全、快速问答 |
| **Claude Sonnet 4.6** | 平衡性能与速度 | 日常开发、代码重构 |
| **Claude Opus 4.6** | 最强推理能力 | 复杂架构设计、深度分析 |

## Cursor 与 Claude Code 对比

### 核心差异

| 维度 | Cursor IDE | Claude Code |
|------|------------|-------------|
| **定位** | AI 原生 IDE | CLI AI 编程助手 |
| **交互方式** | 图形界面，多面板 | 命令行，对话式 |
| **工作流** | 实时代码编辑 | 基于任务的自主 Agent |
| **文件操作** | 手动选择文件 | 自动搜索和编辑 |
| **上下文** | IDE 当前打开的文件 | 整个项目代码库 |
| **Git 集成** | 可视化操作 | 命令行驱动 |

### 详细对比

#### Cursor 的优势

- **可视化体验**：多面板设计，Chat + Edit + Preview 并行
- **实时反馈**：逐行显示 AI 生成过程，可中途打断
- **精细控制**：精确选择代码块进行编辑
- **调试友好**：内联错误修复、测试运行
- **学习曲线低**：类似 VS Code 的操作习惯

#### Claude Code 的优势

- **自主 Agent**：可执行复杂的多步骤任务
- **全局上下文**：理解整个项目结构
- **命令行原生**：与 Git、终端工具无缝集成
- **计划模式**：复杂任务的先规划后执行
- **可编程**：支持自定义 Skills、Hooks、Agents

## 互补使用场景

### 场景一：日常编码 vs 项目重构

```
┌─────────────────────────────────────────────────────────────┐
│                    日常开发流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Cursor IDE                                                  │
│  ├─ 编写新函数时获取智能补全                                 │
│  ├─ 快速修复语法错误                                         │
│  ├─ 解释选中代码片段的含义                                   │
│  └─ 实时对话讨论实现方案                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    重构/迁移流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Claude Code                                                 │
│  ├─ 跨多文件的重构任务                                       │
│  ├─ 依赖关系分析和更新                                       │
│  ├─ 批量测试生成和验证                                       │
│  └─ Git commit 历史分析                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**实践建议**：
- 日常在 Cursor 中编写代码
- 遇到需要跨多个模块的重构时，切换到 Claude Code

### 场景二：问题探索 vs 解决方案实施

```
探索阶段 → Cursor
─────────────────
• 快速尝试不同实现方式
• 通过对话理解问题
• 实时看到代码变化

实施阶段 → Claude Code
─────────────────────
• 生成完整的测试套件
• 执行多步骤验证流程
• 确保 80%+ 测试覆盖率
```

### 场景三：学习新代码库 vs 构建新功能

| 任务 | 推荐工具 | 原因 |
|------|----------|------|
| 阅读和理解新代码库 | **Cursor** | 可以逐行提问，高亮理解 |
| 快速原型开发 | **Cursor** | 实时编辑和迭代 |
| 添加新功能（多文件） | **Claude Code** | 自动规划和执行 |
| 编写测试用例 | **Claude Code** | TDD 工作流内置 |
| 代码审查 | **Claude Code** | 自动化 review agent |
| 性能优化 | **Claude Code** | 全局分析和重构 |

### 场景四：典型协作工作流

```
1. 需求分析
   └─> Cursor Chat：讨论需求，澄清细节

2. 快速原型
   └─> Cursor Edit：快速实现核心功能

3. 测试覆盖
   └─> Claude Code：使用 tdd 生成测试和代码编写

4. 代码审查
   └─> Claude Code：使用 code-reviewer 检查代码质量

5. Git 提交
   └─> Claude Code：生成规范的 commit message
```
