# ruyi包管理器新闻
执行`ruyi update`命令刷新本地软件源时，Ruyi包管理器 会自动检查并列出所有未读新闻，这是最常用的被动获取通知的方式。每次同步源时，系统都会优先展示未读内容，确保你不会遗漏重要信息。也可执行`ruyi news list`查看全部新闻，未读内容会在支持的终端中以绿色高亮显示。
```bash
$ ruyi news list -h           # 查看 list 子命令的帮助文档，获取完整参数说明
$ ruyi news list              # 列出所有新闻
$ ruyi news list --new        # 仅列出未读新闻

$ ruyi news read -h           # 查看 read 子命令的帮助文档，获取完整参数说明
$ ruyi news read 1            # 阅读序号（ID）为 1 的新闻
$ ruyi news read              # 阅读全部未读新闻
$ ruyi news read --quiet      # 不输出任何东西，只标记未读新闻为已读
```
## 新闻阅读状态存储机制
Ruyi 会在本地文件中存储已读新闻的标题，以此来区分已读 / 未读状态，具体路径规则如下：
- 默认路径：`~/.local/state/ruyi/news.read.txt`
- 自定义路径：如果系统配置了 `XDG_STATE_HOME` 环境变量，则文件路径为 `$XDG_STATE_HOME/ruyi/news.read.txt`
