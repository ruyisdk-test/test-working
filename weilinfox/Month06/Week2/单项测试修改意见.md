# RVSPOC S2310 快速上手 ncnn 移植和优化 的复现步骤和修改意见

## 修改意见

ppt p16 有两处修改意见：

1. 在 ``mkdir -p build-duo`` 前应当 ``cd ncnn``
2. 构建 ncnn 依赖 cmake ，这个构建工具通常不会被预装，有必要提前 ``apt-get install cmake``

ppt p19 有四处修改意见：

1. 依赖了 build-duo 目录，需要注明这部分操作需要在 ncnn 目录下进行
2. ``mkdir nanodet`` 应该改为 ``mkdir nanodet-test``
3. ``test.jpg`` 的由来在代码中没有体现，如添加 ``wget https://images.unsplash.com/photo-1623387641168-d9803ddd3f35 -O test.jpg``
4. ``nanodet`` 在默认镜像稳定复现内存不足

ppt p21 有一处修改意见：

1. 在 ``docker run`` 前添加 ``cd duo-buildroot-sdk`` ，否则无法与下文 ``cd /home/work/`` 对应

## 完整复现步骤

### 准备 Milkv-Duo

下载安装最新 [BalenaEtcher](https://github.com/balena-io/etcher/releases/tag/v1.18.13)

下载并解包最新 Milkv-Duo 镜像 [Duo-V1.0.6](https://github.com/milkv-duo/duo-buildroot-sdk/releases/tag/Duo-V1.0.6)

使用 BalenaEtcher 烧写镜像到 Micro SD 卡即可。

上电等待蓝灯闪烁，主机应当可以连到网络。

### 获取源码

```bash
git clone https://github.com/Tencent/ncnn.git
export RISCV_ROOT_PATH=$(pwd)/duo-sdk/riscv64-linux-musl-x86_64/
cp ncnn/toolchains/c906-v240.toolchain.cmake ncnn/toolchains/c906-v240-musl.toolchain.cmake

vim ncnn/toolchains/c906-v240-musl.toolchain.cmake
    set(CMAKE_C_COMPILER "${RISCV_ROOT_PATH}/bin/riscv64-unknown-linux-musl-gcc")
    set(CMAKE_CXX_COMPILER "${RISCV_ROOT_PATH}/bin/riscv64-unknown-linux-musl-g++")
```

### 进行构建

```bash
sudo apt-get install cmake
cd ncnn
mkdir -p build-duo
cd build-duo
cmake -DCMAKE_TOOLCHAIN_FILE=../toolchains/c906-v240-musl.toolchain.cmake \
      -DCMAKE_BUILD_TYPE=release -DNCNN_OPENMP=OFF -DNCNN_THREADS=OFF -DNCNN_RUNTIME_CPU=OFF -DNCNN_RVV=ON \
      -DNCNN_SIMPLEOCV=ON -DNCNN_BUILD_EXAMPLES=ON ..
cmake --build . -j 12
```

sudo docker exec -it duodocker /bin/bash
