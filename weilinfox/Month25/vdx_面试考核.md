# vdx 面试考核

考核时间 2025/07/08-2025/07/20

考核题目需要产出代码并正确提交 pr 到对应的仓库， pr 将被 review，优雅地提交 pr 也是考核的一部分，有疑问及时沟通。注意已提交的 pr 的状态，若其他贡献者 pr 的合并导致了你的 pr 出现 conflict，请及时处理。若提前完成考核题可以用余下的时间寻找题目以外的贡献点，若考核时间不足以做完全部考核题则在有限的时间内尽可能展示自己。若后续面试通过，考核题目也将计入后续实习产出，作为工资的计算依据。

面试方式为腾讯会议，需要开麦和共享屏幕，不需要开摄像头。需要有一个能用的 Linux 环境来解一些面试期间提出的问题，发行版不限，但曾经出现使用 WSL 面试导致一些操作不能完成的情况。整个面试过程可以 Google 也可以 ChatGPT，但是这些使用都以共享屏幕为前提，当然我们推荐不使用。面试时长会尽量控制在 1 小时以内。

1. 优化 [ruyisdk.org](https://ruyisdk.org/) 页面（在达到字面要求的同时尽可能让页面达到整体的美观和协调）
    由于官网翻新工作仍在进行中，其中首页和施工中的[社区](https://github.com/ruyisdk/ruyisdk-website/pull/160)页面为新页面，其他页面若有全部或部分重新设计的必要则与这两个页面看齐。
    注意桌面版和移动版均需要有较好的效果，特别是移动版；注意桌面版页面缩放造成的元素异常；注意移动版老年模式造成的页面异常
    + “关于”（/contact）页面当前存在而需要修复的一些问题：
        1. 微信公众号二维码大小在页面中不协调
        2. QQ 群组件 ``<QQGroupList />`` 为暗色底色，与整体的亮色主题不协调
        3. QQ 群组件组件在页面中不协调
    + 首页和页脚分别有两处 RevyOS 的链接可以重定向到 docs.revyos.dev
        1. RevyOS 的页面默认为中文，但是也支持[英文](https://docs.revyos.dev/en/)，而 ruyisdk.org 只会跳转到默认首页
        2. 在 ruyisdk.org 首页的英文和德文页面中使用 RevyOS 网站的英文链接
    + 页脚的“微信公众号”和“QQ群”被设计为鼠标悬停时悬浮展示二维码，但这个功能尚有 bug
        1. 与中文页面相比，在英文和德文的页面中，这两个二维码是错位的
        2. 移动端的行为很奇怪
    + 数据统计页面
        1. 修复数据统计页面中不合理的设计，比如“分目录的下载数量”并不适合用柱状图展示，可以将这部分数据放在“RuyiSDK 数据纵览”这一块的下方，也可以有其他设计
        2. 该页面桌面端浏览器 F12 打开控制台时，元素会被挤压，比如“数据更新时间”被挤压到页脚下方，柱状图元素混乱
        3. 注意“最常用包”由于 API 暂未返回统计数据，“暂无数据”是正常的
2. 为 [ruyisdk.org](https://ruyisdk.org/) 添加新的页面功能（实现一个较为复杂的功能性组件）
    用过 ``ruyi device provision`` 吗？这是一个镜像刷写器，首先安装并试用
    + 在网页中实现一个 ``ruyi device provision`` 演示试用组件，要求输出效果和 Ruyi 包管理器的实际输出一致
    + 由于 ``ruyi device provision`` 中的列表可能会更新，设计要求数据方便更新
    + 不强制要求键盘输入，可以以 URL 参数的方式代替
    + 如果便于集成进现有页面，则可以以“博客”的形式提交
    + 如果不便于集成进现有页面，也可以单开仓库暂作技术上的实现，之后再讨论合入方式
